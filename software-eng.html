Chapter 7: Software Engineering
TERM DEFINITION
software engineering branch of computer science that discovers principles to develop software systems
integrated development environment (IDE) software application that contains a variety of tools for developing software
Association for Computing Machinery (ACM)
  • purpose of ACM - to advance the arts, sciences, and applications of computers internationally
  • benefits of ACM membership - online educational resources, a full-year subscription, targeted job offers, and networking opportunities
  • student ACM membership - similar benefits as professional members, but cannot vote for ACM officers
Institute of Electrical and Electronics Engineers (IEEE)
  • purpose of IEEE - to support innovation and excellence in technology
  • benefits of IEEE membership - digital library, professional education online courses, IEEE and affinity program discounts, and networking opportunities
  • student IEEE membership - similar to benefits as professional members
Software Life Cycle: Development phase
  • Requirements analysis - identifies services, conditions, and interactions for the system
  • Design - the blueprint for the system
  • Implementation - creates programs, data files, and databases for the system
  • Testing - debugging and confirming software requirements are met
modular implementation division of software into manageable units called modules
Unified Modeling Language (UML) variety of tools for the object-oriented paradigm
use case an interaction by a user with a system
use case diagram a UML tool looks at an image of a system from the user's viewpoint
Date relationships:
  • one-to-one - one class is paired with only one other class
  • one-to-many - one class is paired with many classes
  • many-to-many - many classes are paired with many different classes
design pattern a solution for a recurring problem when designing software
Software quality assurance (SQA) oversee and enforce quality control systems by approving the requirements or design of software
Software testing:
  • glass box testing - from a developer's perspective
  • black box testing - from the user's perspective
  • beta testing - performed by a subset of users before the release of the software
  • alpha testing - performed by developers before beta testing
Documentation:
  • user documentation - explains software features with instructions about how to use them
  • system documentation - explains the composition of software in order to help maintain it
  • technical documentation - provides installation and service information for the software
Intellectual property law
  • software license - a legal agreement that describes how the user can use the software without infringing on the rights of the developer
  • software disclaimer - describes liability limitations of a software developer
  • best defenses against software liability claims - Sound software engineering principles and accurate record-keeping during development can be used to ensure the level of care suits the software's application.
