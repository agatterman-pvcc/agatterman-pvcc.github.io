All About GitHub

  1. What is the version control system?
    • A system that records changes to a file or set of files over time for future reference.

  2. What is Git?
    • A Distributed Version Control System(DVCS) that is used to save different versions of a file or set of files so that any version is retrievable.

  3. What is GitHub?
    • A web-based platform that allows users to host Git repositories.

  4. What is the difference between Git and GitHub?
    • GitHub hosts Git repositories online while Git manages multiple versions of a file.

  5. Who started Github and how was it started?
    • Tom Preston-Werner and Chris Wanstrath started GitHub in 2007 because programmers needed a service where they could collaborate.

  6. What company owns it now?
    • Microsoft acquired GitHub in 2018.

  7. How much does a GitHub account cost?
    • GitHub is free for individuals and organizations.

  8. What is Octocat?
    • GitHub's mascot. This mascot is a reference to Git's ability to merge 2 or more branches of development at the same time known as "octopus merge."


Git/GitHub terms:

  • repository
    • A collection of a project's files and each file's revision history

  • commit
    • An individual change to a file

  • fork
    • A personal copy of another user's repository

  • push
    • Committed changes to a repository on GitHub

  • pull requests
    • Proposed changes to a user's repository by a repository's collaborators

  • workflows
    • A configurable automated process that will run one or more jobs

  • issues
    • Suggested improvements, tasks, or questions to a repository

  • raw button
    • Opens a file without any styling

  • blame button
    • Shows when a line in a file was modified and who modified it
